* reMarkable Templates

A toolkit for authoring, browsing, and rendering custom [[https://remarkable.com][reMarkable]] tablet page templates.

** Project Structure

#+begin_src
remarkable_templates/
├── python/          # Experimental Python prototype (template I/O utilities)
├── templates_orig/  # Device-original .template files (source of truth, git-ignored)
└── web/             # React/TypeScript browser app (primary development target)
#+end_src

** Template Format

Templates are JSON files (~.template~) that describe a notebook page layout as a
tree of graphic items. Each file contains:

- ~constants~ — named scalar values evaluated in order; later entries may reference
  earlier ones. Values may be numbers or arithmetic/ternary string expressions.
- ~items~ — a tree of ~path~, ~text~, and ~group~ items that define the visual content.

Groups support a ~repeat~ field for tiling their children across the page:

| Value       | Behaviour                                                        |
|-------------|------------------------------------------------------------------|
| ~0~         | Render once at tile origin (no repeat)                           |
| ~N~         | Render exactly N tiles                                           |
| ~"down"~    | Fill downward from tile origin to viewport bottom                |
| ~"right"~   | Fill rightward from tile origin to viewport right edge           |
| ~"up"~      | Fill upward from tile origin to viewport top                     |
| ~"infinite"~| Fill in both directions to cover the full viewport               |
| ~"expr"~    | Any constant expression resolving to a number (e.g. ~"columns"~) |

** Supported Devices

| ID     | Name                    | Portrait       | Landscape      |
|--------|-------------------------|----------------|----------------|
| ~rm1~  | reMarkable 1            | 1404 × 1872 px | 1872 × 1404 px |
| ~rm2~  | reMarkable 2            | 1404 × 1872 px | 1872 × 1404 px |
| ~rmPP~ | reMarkable Paper Pro    |  954 × 1696 px | 1696 ×  954 px |

Templates that behave differently on smaller screens use ~mobileMaxWidth~ (typically
1000 px) in ternary constant expressions to branch between layouts.

** Web App

The primary interface for browsing and validating templates.

#+begin_src bash
cd web
pnpm install
pnpm dev        # → http://localhost:5173
pnpm test       # run all unit tests
pnpm build      # production build
#+end_src

See ~web/README.md~ for full details.

** Adding Templates

1. Place the ~.template~ JSON file in ~web/public/templates/~
2. Add a registry entry to ~web/public/templates/templates.json~
3. The browser will pick it up on next page load — no rebuild required

** Python Prototype

~python/~ contains early exploration code for reading and writing ~.template~ files.
It is not a dependency of the web app and may be out of sync with the current
template format.
